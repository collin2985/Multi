CLAY NATURAL OBJECT IMPLEMENTATION PLAN
========================================

RESEARCH FINDINGS:
------------------

Natural objects work through several integrated systems:

1. PHYSICS & COLLISION SYSTEM
   - Objects with 'radius' in CONFIG.CONSTRUCTION.GRID_DIMENSIONS get cylindrical colliders
   - Collision group: COLLISION_GROUPS.NATURAL
   - Physics manager creates static colliders automatically when objects spawn
   - Spatial queries detect nearby colliders for proximity detection

2. PROXIMITY DETECTION SYSTEM
   - game.js maintains activeProximityObjects Map populated from physics spatial queries
   - Calculates distances to player, categorizes objects (trees/rocks/structures)
   - Sets gameState.nearestObject with object data + toolCheck
   - rockTypes array (line 1447 in game.js) determines what's considered a rock

3. UI INTERACTION SYSTEM
   - ui.js reads gameState.nearestObject
   - rockTypes array (line 163 in ui.js) controls UI button display
   - Shows "Mine [Rockname]" button when: isInChunk, hasRequiredTool, hasResources, !cooldown, !moving

4. HARVESTING SYSTEM
   - game.js onRemoveObject callback checks isRock (line 395) to start harvest
   - Calls startHarvestAction with 'stone' harvestType
   - ResourceManager.js requires 'pickaxe' tool (line 490)
   - Creates inventory item matching rock name (limestone -> limestone item)

5. RESOURCE TRACKING
   - objects.js sets totalResources and remainingResources on spawn (line 593)
   - Rocks give 1 resource when mined
   - Resource depletion tracked in userData

6. MARKET SYSTEM
   - config.js defines buyPrice/sellPrice/min/max quantities (line 487)
   - server/MessageHandlers.js initializes market inventory with quantities and qualityAverages
   - Two locations: structureChange.inventory (line 355+376) and addedPayload.inventory (line 443+464)

7. TERRAIN GENERATION
   - terrain.js has defaultMinDistances for placement restrictions (line 1559)
   - Prevents placing objects too close to natural features
