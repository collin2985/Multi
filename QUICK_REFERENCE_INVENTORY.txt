DATA STRUCTURES
===============

Item Format:
{
    id: "type_timestamp_random",
    type: "limestone",
    x: 0, y: 0,              // grid position
    width: 1, height: 1,     // grid size
    rotation: 0,             // 0 or 90
    quality: 75,             // 0-100
    durability: 100          // tools only
}

Inventory Format:
{
    rows: 10,
    cols: 5,
    items: [...],
    slotSize: 60,
    gap: 2
}

Chunk Storage Format (server):
{
    "objectChanges": [
        {
            "action": "add",
            "id": "crate_123",
            "inventory": {
                "rows": 10,
                "cols": 10,
                "items": [...]
            }
        }
    ]
}

INVENTORY SIZES
===============
Backpack: 10 rows x 5 cols = 50 slots
Crate: 10 x 10 = 100 slots
House: 10 x 10 = 100 slots
Garden: 2 x 2 = 4 slots
Market: 10 x 10 = 100 slots

KEY FUNCTIONS
==============

PlayerInventory.js:
  addItem(item) - Add to grid
  removeItem(itemId) - Remove by ID
  isPositionFree(x, y, w, h) - Check collision
  serialize() / deserialize() - Save/load

InventoryUI.js:
  renderInventory() - Draw backpack
  renderCrateInventory() - Draw storage
  renderBackpackItem(item, container) - Render single item
  onMouseUp(event) - Handle item placement
  _handleCrateModePlacement(item, source) - Transfer between inventories
  showTooltip(event, item) - Display item info

GridUIHelpers.js:
  calculateGridSize(rows) - Compute slot pixel size
  gridToPixel(x, y, slotSize, gap) - Convert grid to pixels
  pixelToGrid(x, y, slotSize, gap) - Convert pixels to grid
  calculateItemSize(w, h, slotSize, gap) - Item display size

NETWORK MESSAGES
================

Client sends:
  get_crate_inventory {crateId, chunkId}
  save_crate_inventory {crateId, chunkId, inventory}

Server responds:
  crate_inventory_response {crateId, inventory}
  crate_inventory_updated {crateId, inventory}

Server handlers:
  server/MessageHandlers.js:
    handleGetCrateInventory(ws, payload)
    handleSaveCrateInventory(payload)

Client handlers:
  public/network/MessageRouter.js:
    handleCrateInventoryResponse(payload)
    handleCrateInventoryUpdated(payload)

ITEM QUALITY & DURABILITY
==========================

Quality (0-100):
  Affects durability loss
  Higher quality = less durability loss
  Formula: durabilityLoss = Math.ceil(100 / quality)

Durability (tools only):
  Starts at 100
  Decreases per use:
    - Axe: -10
    - Saw: -10
    - Pickaxe: -10
    - Hammer: -5
    - Chisel: -Math.ceil(100/stoneQuality)

COLOR CODING:
  Green (>= 80): Good quality
  Yellow (>= 40): Medium quality
  Red (< 40): Poor quality

DRAG & DROP MECHANICS
=====================

Start: onItemMouseDown()
  - Save original position
  - Store mouse position
  - Setup global listeners

During: onMouseMove()
  - Update mouse coordinates
  - Determine target (backpack or crate)
  - Update ghost item position
  - Re-render inventory

Rotation: Press 'R'
  - Toggle rotation 0 <-> 90
  - Update image (use Rtype.png for rotated)

End: onMouseUp()
  - Validate new position
  - If valid: update item x/y
  - If invalid: restore position
  - Mark crate dirty if moved
  - Clean up listeners
  - Re-render

PERSISTENCE
============

Client-side:
  Backpack: gameState.inventory (in-memory only)
  Crate: crate.userData.inventory (cached after load)

Server-side:
  Format: ./public/chunk_X,Y.JSON
  Memory cache: ChunkManager.chunkCache (Map)
  Load/Save: ChunkManager.loadChunk() / saveChunk()

Auto-save: When player closes inventory (dirty flag)
